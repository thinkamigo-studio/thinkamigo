**On the brief document**

- Append-only. Nothing removed, condensed or rewritten without explicit instruction. 
- State this rule at the top of every session and every segment file. DONE
- Break into segments to contain errors — same principle as the CSS architecture.
- The brief is a primary source. Treat it like an archive.

**On working with Claude**

- Start fresh sessions for major document work.
- Long conversations degrade. Context gets compressed, earlier instructions deprioritised.
- Never ask Claude to rebuild from scratch — always update from the existing base document.
- Paste only the relevant segments for the session in hand.
